# 发布与部署

这个项目是静态文档项目，可以用 GitHub Pages、GitBook、Docsify 或其他静态网站服务发布。

## 推荐方式：GitHub Pages + docs 目录

项目已经准备了 `docs/index.html`，可以直接把 `docs` 目录作为站点根目录。

基本流程：

1. 创建 GitHub 仓库。
2. 上传本项目所有文件。
3. 在仓库设置中找到 Pages。
4. 选择从分支发布。
5. 目录选择 `docs`。
6. 保存后等待页面生成。

如果页面样式没有加载，优先检查：

- `docs/index.html` 是否存在。
- `docs/SUMMARY.md` 是否存在。
- GitHub Pages 是否选择了 `docs` 目录。
- 浏览器是否能访问外部 CDN。

## 本地预览

可以直接打开：

```text
docs/index.html
```

如果浏览器限制本地文件加载，建议使用一个本地静态服务器打开 `docs` 目录。

## 发布前检查

- 首页能打开。
- 左侧目录能显示。
- 每个目录链接都能跳转。
- 中文没有乱码。
- 西语重音符号显示正常。
- README、LICENSE、CONTRIBUTING 都存在。
- 没有引用本机私有路径。

## 版本命名建议

- v0.1：项目骨架和核心章节。
- v0.2：练习册和资源索引。
- v0.3：考试专题。
